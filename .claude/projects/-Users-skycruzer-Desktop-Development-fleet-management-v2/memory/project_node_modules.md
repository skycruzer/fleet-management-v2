---
name: iCloud sync corrupts node_modules
description: macOS iCloud Drive creates duplicate directories with spaces in node_modules, causing build failures and I/O hangs
type: project
---

iCloud Drive syncing Desktop creates hundreds of duplicate directories with spaces (e.g., `react 2`, `next 2`, `@swc 3`) inside `node_modules`. This causes npm install failures (ENOTEMPTY), build lock contention, and kernel-level I/O hangs.

**Why:** iCloud syncs the Desktop folder where the project lives, and node_modules has thousands of files that trigger sync conflicts.

**How to apply:** If builds or npm operations hang, check for space-named duplicates in node_modules. Fix: move node_modules to /tmp, run `npm ci`. Long-term: exclude node_modules from iCloud or move project off Desktop.
